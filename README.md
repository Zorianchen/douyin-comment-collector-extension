# Douyin Comment Collector

抖音评论区采集 Chrome / Edge 浏览器插件。

当前版本：`0.3.6`

## v0.3.6 更新

- **修复到底漏采**：看到“暂时没有更多评论”时不再立刻停止，会先补采当前屏幕底部可见评论，再结束。
- **增强评论区点击**：点评论区按钮会同时尝试右侧评论气泡、顶部评论 tab、`#semiTabcomment` 和固定 XPath 多个目标。
- **模拟真实点击**：点击时同时触发 pointer / mouse / click 事件，并对元素中心点二次点击，提高抖音页面响应概率。

## v0.3.5 更新

- **强化去重规则**：同一视频下同一用户同一评论文本只保留一条，不再因为父评论字段、URL 参数或 DOM 节点不同重复导出。
- **导出时再次去重**：CSV / JSON 导出前会重新强去重，优先保留信息更完整的记录。
- **统一 popup 与悬浮面板去重**：浏览器插件弹窗导出和页面悬浮面板导出使用同一类规范化规则。

## v0.3.4 更新

- **出现“暂时没有更多评论”即停止**：识别页面底部没有更多评论提示后自动结束采集，不再空跑。
- **悬浮面板默认缩小**：页面加载后默认显示一个小圆形「评」图标，点击后展开完整控制面板。
- **悬浮面板新增设置**：直接在悬浮面板里填写最大评论数和滚动间隔，不用打开插件弹窗。
- **悬浮面板新增导出**：支持在页面内直接导出 CSV / JSON。
- **修复“点评论区”无反应**：点击 XPath 命中的 span 时会转到可点击父级，并连续触发两次点击，提高生效概率。

## v0.3.3 更新

- **按固定 XPath 点击评论 tab**：优先使用 `//*[@id="semiTabcomment"]/span` 定位右侧「评论」，避免误点「详情」。
- **不再采集详情页内容**：如果没有识别到真实评论列表，插件会停止并提示，不会把详情区内容当成评论。
- **锁定右侧评论面板**：评论列表、长评论展开、二级回复展开、滚动和监听都优先限定在评论 tab 下方的右侧面板。
- **彻底禁止兜底滚动视频页**：找不到评论容器时不再滚动页面，减少视频跟着滚的问题。

## v0.3.2 更新

- **修复误点“详情”**：打开评论区时不再泛化点击任意“评论”文字，只识别右侧顶部标签栏中尺寸和位置更像 tab 的“评论”。
- **减少启动误切 tab**：开始采集时如果已经识别到评论容器，就不再额外点击标签，避免从评论区被切回详情页。
- **如果评论按钮没识别到**：悬浮面板会提示手动点击右侧顶部“评论”标签后再开始。

## v0.3.1 更新

- **页面内悬浮采集面板**：打开抖音视频页后，页面右上角会出现一个可拖动悬浮面板，不用反复打开浏览器插件弹窗。
- **支持拖动位置**：按住悬浮面板顶部可拖动，位置会保存到本地。
- **一键点击评论区**：悬浮面板新增「点评论区」按钮，会尝试点击页面右侧的“评论”标签，适配用户截图里的位置。
- **采集完成提示**：达到最大评论数或连续多轮无新增时，会在页面内弹出提示，显示当前采集条数。
- **悬浮面板控制**：支持开始、暂停、继续、停止、去重、清空和折叠。

## v0.3.0 更新

- **不再主动滚动视频页面**：采集时优先锁定评论区滚动容器，只有完全找不到评论容器时才兜底滚动页面，减少视频被带着滚走的问题。
- **二级回复默认开启**：默认勾选「采集二级回复，自动展开」，并放宽「查看全部 N 条回复 / 展开更多回复」等按钮识别。
- **长评论展开增强**：识别 `aria-label`、`title`、`role=button` 等展开控件，优先在评论卡片内处理带 `...` / `…` 的文本，点击失败时允许后续重试。
- **过滤纯省略号评论**：采集时不再把 `...` / `…` 当成有效评论保存，尽量等待展开后的完整文本。

## v0.2.9 更新

- **评论 ID 优先去重**：导出字段新增 `commentId`、`domId`，采集中优先使用页面评论 ID / DOM ID 去重，减少展开前后和滚动重复。
- **更稳的到底判断**：综合本轮新增、文本更新、展开动作、`scrollTop`、`scrollHeight` 和连续稳定轮数判断，不再轻易提前停止。
- **增强顶部回滚**：开始采集前多轮尝试定位评论区并回到顶部，尽量从第一条评论开始抓。
- **弹窗进度反馈**：新增轮次、本轮新增、连续无新增、监听状态显示，方便判断插件是否仍在工作。

- **减少漏采**：保留「监听 + 主动滚动」双模式，页面新增评论节点会实时解析，滚动扫描继续兜底。

## 安装步骤

1. 解压 `douyin-comment-extension-v0.3.6.zip`。
2. Chrome：打开 `chrome://extensions/`；Edge：打开 `edge://extensions/`。
3. 开启「开发者模式」。
4. 点击「加载已解压的扩展程序」，选择解压后的 `douyin-comment-extension` 文件夹。
5. 安装成功，浏览器工具栏会出现插件图标。

## 使用方法

1. 打开 Chrome / Edge，登录抖音网页版。
2. 打开任意抖音视频详情页。
3. 点击浏览器工具栏的插件图标。
4. 点击「检测页面」确认页面识别正常。
5. 配置采集参数：
   - 最大评论数（默认 500）
   - 滚动间隔毫秒（默认 2500）
   - 是否采集二级回复（默认开启）
   - 是否展开长评论文本（默认开启）
6. 点击「开始评论」，插件自动滚动评论区并采集。
7. 采集完成后点击「导出评论 CSV」或「导出评论 JSON」。

## 导出字段

- `videoUrl`：视频/图文链接
- `commentId`：页面中识别到的评论 ID，可能为空
- `domId`：评论 DOM 节点 ID，可能为空
- `userName`：用户昵称
- `userProfileUrl`：用户主页链接
- `commentText`：评论文本
- `likeCount`：点赞数
- `replyCount`：回复数
- `commentTime`：评论时间
- `level`：评论层级（main 主评论 / reply 二级回复）
- `parentUserName`：父评论用户昵称
- `parentCommentText`：父评论文本
- `replyToUserName`：回复对象用户昵称
- `commentImageUrls`：评论内图片链接
- `collectedAt`：采集时间
- `source`：数据来源标识

## 技术说明

- 平台：Chrome / Edge Manifest V3
- 架构：content script 注入页面，chrome.storage.local 本地存储，Blob 下载
- 数据：仅保存在本地浏览器，不上传服务器
- 权限：`activeTab`、`scripting`、`storage`、`downloads`，仅匹配抖音域名

## 注意事项

- 本插件仅采集页面上公开展示的评论内容。
- 不保存账号密码，不自动登录，不绕过验证码或风控。
- 建议先点「检测页面」确认评论区识别正常后再开始采集。
- 如果长评论仍有截断，确认「展开长评论文本」已勾选，或检查页面是否还有其他展开按钮样式。
